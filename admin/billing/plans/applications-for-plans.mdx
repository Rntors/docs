title: "Applications For Plans         "
---

description: "Set up plan sign-up requests to collect membership applications and verify customers"
---------------------------------------------------------------------------------------------------

With sign-up requests, users can apply for membership plans, and you can vet them before creating a subscription.

Plan sign-up requests are ideal for:

* Niche or curated spaces that require personal or professional verification
* Verification-based plans that require ID uploads or supporting documents
* Discounted plans for students, nonprofits, or other groups that need eligibility confirmation.

<Info>
  Plan sign-up requests are available only on [specific plans](https://www.spacebring.com/docs/billing/plans#sign-up-setting).
</Info>

<Tip>
  For plans that don't require verification, [use the **Immediate** sign-up type](https://www.spacebring.com/docs/billing/plans/immediate-sign-ups) instead.
</Tip>

## Configure plan sign-up requests

1. [Create](https://www.spacebring.com/docs/billing/plans#set-up-plans) or [edit](https://www.spacebring.com/docs/billing/plans#edit-or-delete-a-plan) a plan.

2. Enable **Sign-up**.

3. Select **Request** as the sign-up **Type**.

<Frame>
  <img
    src="/images/placeholders/applications-plan-request-setting.png"
    alt="Edit plan screen showing Sign-up enabled with Request selected as the sign-up type"
  />
</Frame>

4. [Enable the **Support** page](https://www.spacebring.com/docs/support/tickets) and set it to **Public** visibility so users can see administrators’ replies.

<Warning>
  If the **Support** page is disabled, the `Apply` button will be greyed out on plans. If the **Support** page is not set to **Public** visibility, users can apply for plans but won’t see administrator responses.
</Warning>

## How users apply for a plan

1. Customers press `Sign up` next to their desired location to view available plans.

<Frame>
  <img
    src="/images/placeholders/applications-public-plans.png"
    alt="Customer portal showing the Sign up button and available membership plans"
  />
</Frame>

2. Select a plan, set their preferred subscription start date, renewal, quantity, and whether they want to sign up as an individual member or as a business.

3. Press `Apply` to submit their request.

<Frame>
  <img
    src="/images/placeholders/applications-plan-request-form.png"
    alt="New subscription application form with start date, renewal, quantity, and business options"
  />
</Frame>

## How administrators manage plan sign-up requests

1. Each plan sign-up request appears as a new ticket in **Support** > **Tickets**.

2. Administrators manage the request through the support ticket:

<Frame>
  <img
    src="/images/placeholders/applications-support-ticket.png"
    alt="Support ticket created from a plan sign-up request"
  />
</Frame>

* Communicate with the applicant
* Request verification documents
* Ask follow-up questions

3. If the user is approved, click the user’s name in the ticket and press `Invite as member`.

<Frame>
  <img
    src="/images/placeholders/applications-invite-as-member.png"
    alt="Customer profile showing the Invite as member action"
  />
</Frame>

4. [Create a subscription](https://www.spacebring.com/docs/billing/subscriptions) for the requested plan for the customer.

5. If the user requested a company subscription, [create a company](https://www.spacebring.com/docs/customer-management/create-a-company) first > [add this user to the company](https://www.spacebring.com/docs/customer-management/create-a-company#add-users-to-a-company) as a manager > create a company subscription.

<Frame>
  <img
    src="/images/placeholders/applications-company-subscription.png"
    alt="Customer profile showing company selection while creating a subscription"
  />
</Frame>

<Tip>
  To avoid missing any notifications about plan sign-up requests, [enable notifications](https://www.spacebring.com/docs/account/notifications) about new support tickets.
</Tip>

## FAQs

<AccordionGroup>
<Accordion title="How do I add a customer who applied for a booking or subscription as a member or nonmember?">

When a user applies for a booking or subscription, they are not a member or a nonmember yet. That’s why they don’t appear on the **Customers** page yet.

1. Click the user’s name from the support ticket to open their profile.

2. Click `Invite as member` or `Add as nonmember` at the top right.
