---
title: "Troubleshoot email delivery issues "
---

Sometimes your members may not receive emails from the member portal or mobile app. Use the tips below to troubleshoot delivery issues with invitations, invoices, and login codes.

## Invitations to join

If a member can't find their email invitation, open their profile in [**Customers**](https://app.rntor.com) and use the invitation action to send it again. RNTOR sends invitations from `noreply@rntor.com`.

<Frame>
  ![Invitation email troubleshooting](/images/troubleshoot-email-invitation-error.png)
</Frame>

### If you see an error, follow these steps

<Steps>
  <Step title="Check the email address">
    Make sure the invitee's email is spelled correctly. If there's a typo, [remove the user](/admin/customer-management/users/remove-users) and re-invite them with the correct email.
  </Step>
  <Step title="Ask the user to allow RNTOR emails">
    Some corporate inboxes block messages from new senders. Ask the user (or their IT team) to allow emails from `noreply@rntor.com`.
  </Step>
  <Step title="Resend the invitation">
    From the user's profile in **Customers**, send the invitation again. The user has 24 hours to follow the link before going through the standard login flow.
  </Step>
</Steps>

### If you do not see an error, try these tips

- Ask the user to check **Spam**, **Promotions**, and other inbox filters.
- Confirm they're checking the same email address you invited.
- Ask them to allow emails from `noreply@rntor.com`.
- If the message still doesn't arrive, try an alternative address (for example `@gmail.com` or `@outlook.com`).

## Invoices

If a member can't find an invoice email, try the following:

<Frame>
  ![Invoice email troubleshooting](/images/troubleshoot-email-invoice.png)
</Frame>

- Ask the user to check the inbox of the email they registered with. They can verify it on their [**Account**](/member/web-portal/account) page in the member portal.
- Double-check that the invoice was sent. Open the invoice in [**Billing → Invoices**](/admin/billing/invoices) and use [**Send an invoice**](/admin/billing/invoices/send-an-invoice) to deliver it again.
- For company invoices, make sure a [company manager](/admin/customer-management/companies/company-members-and-managers) is set and has a valid email address.

## Login codes

If a user isn't receiving the one-time login code by email, ask them to do the following:

<Frame>
  ![Login code troubleshooting](/images/troubleshoot-email-login-code.png)
</Frame>

<Steps>
  <Step title="Check Promotions and Spam">
    Login codes can be filtered into **Promotions** or **Spam**. Ask the user to check those folders.
  </Step>
  <Step title="Allow RNTOR emails">
    Ask their email provider (or IT team) to allow messages from `noreply@rntor.com`.
  </Step>
  <Step title="Use the mobile app">
    If the user is already signed in on another device, they can approve the login from the push notification on that device.
  </Step>
</Steps>

## Improve email deliverability

RNTOR follows email best practices so messages land in the inbox. Delivery can still be affected by user-side inbox filters that block external senders. Use the recommendations below to improve deliverability for your members.

<Frame>
  ![Email deliverability](/images/troubleshoot-email-deliverability.png)
</Frame>

### Whitelist the email address, add it to contacts, or mark an email as "Not spam"

If members find RNTOR messages in **Spam**, ask them to whitelist `noreply@rntor.com` in their email provider.

- Instructions for [Gmail](https://support.google.com/mail/answer/1366858)
- Instructions for [Outlook](https://support.microsoft.com/en-us/office/add-recipients-of-my-email-messages-to-the-safe-senders-list-be1baea0-beab-4a30-b968-9004332336ce)
- Instructions for [Yahoo](https://help.yahoo.com/kb/SLN28056.html)

### Customize the email footer

You can personalize the footer of RNTOR emails to include your business information — for example, your company name, address, or legal disclaimer. This keeps your emails professional and compliant.

A platform administrator can update the footer text from **Platform settings → Emails → Templates**. If you don't have access, [contact support](/admin/administration/contact-support) and we'll update it for you.

### Configure custom MAIL FROM

By default, RNTOR sends transactional emails from `noreply@rntor.com`. If you want emails to come from your own branded address (for example, `notifications@yourcompany.com`), [contact support](/admin/administration/contact-support) — this is set up by the RNTOR team.
