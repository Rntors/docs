## نظرة عامة

تطبّق Rntor API تحديد المعدل لضمان الاستخدام العادل والحفاظ على جودة الخدمة لجميع المستخدمين.

## حدود المعدل

| الخطة | الطلبات في الدقيقة | الطلبات في اليوم |
|------|---------------------|------------------|
| Standard | 60 | 10,000 |
| Professional | 120 | 50,000 |
| Enterprise | 300 | غير محدودة |

## رؤوس حدود المعدل

تتضمن كل استجابة API رؤوساً لمساعدتك على تتبع استخدامك:

| الرأس | الوصف |
|--------|-------------|
| `X-RateLimit-Limit` | الحد الأقصى للطلبات المسموح بها في الدقيقة |
| `X-RateLimit-Remaining` | الطلبات المتبقية في النافذة الحالية |
| `X-RateLimit-Reset` | الطابع الزمني Unix عند إعادة تعيين الحد |

## مثال على رؤوس الاستجابة

```http
X-RateLimit-Limit: 60
X-RateLimit-Remaining: 45
X-RateLimit-Reset: 1704067200
```

## معالجة حدود المعدل

عند تجاوز حد المعدل، ستتلقى استجابة `429 Too Many Requests`:

```json
{
  "error": {
    "code": "rate_limit_exceeded",
    "message": "Rate limit exceeded. Please retry after 30 seconds.",
    "retry_after": 30
  }
}
```
