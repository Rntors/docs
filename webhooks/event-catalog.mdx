## Subscribing to Events

### Via Svix Portal

1. Navigate to **Settings → Developers → Webhooks**
2. Click on your endpoint (or create one)
3. Under **Event Types**, select the events you want
4. Save changes

### Filtering Events

You can subscribe to:
- **All events**: Receive every webhook type
- **Category**: All events in a category (e.g., all booking.*)
- **Specific events**: Only selected event types

<Tip>
  Start with essential events for your integration. You can always add more later.
</Tip>

## Payload Structure

All events follow the same envelope structure:

```json
{
  // Entity-specific fields
  "id": "uuid",
  "field1": "value",
  "field2": "value",
  
  // Metadata (present in all events)
  "_meta": {
    "event_id": "uuid",
    "event_type": "category.action",
    "merchant_id": "uuid",
    "timestamp": "2024-01-15T10:30:00.000Z"
  }
}
```

See [Payload Examples](/webhooks/payload-examples) for detailed examples of each event type.
